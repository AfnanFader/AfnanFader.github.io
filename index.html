Lab Rover

This is a collaboration project with a final year engineering student, as I was in charge in coding the rover system and communication between the server and the rover. The purpose of this project is to develop a multi axis viewing system for purpose of laboratory exercises. As distant learning or online learning has become prominent in a world were global education improved throughout the years. This project is done to improve the laboratory exercises for undergraduate engineering students.

As the current method is based on watching laboratory videos and using a set of given data to be processed into the laboratory report. This method does not provide same atmosphere as the physical method.

The rover main parts are comprises of two Arduino Uno and a Raspberry Pi 3 Model B as the main unit. Both Arduino communicate with the Pi with serial communication via the USB. Both Arduino has it own functions, such as controlling the movement of the motor, linear actuator, stepper motor and the monitoring of the batttery. All the data is transfered and recieve by the Pi with a bi-directional serial communication. The Pi module has a radio module to recieve movement commands from the server and stream the video feed from the camera via Wi-Fi. The rover is also able to be controlled via Wi-Fi.

Curently, the project is still ongoing and with future plans of adding autonomy to it. Some of the future plans are the following:

1) Replacing the Raspberry Pi with Jetson Nano for accelerated machine learning.

2) Adding multiple camera for collision detection.

3) Use visual odometry and machine learning to improve the rover autonomy capability.

4) Deploy a contact charging/Wireless charging station for the rover.

CariMuka

CariMuka is a product of Surveillance Video Visual Verification (SV3). CariMuka is developed specifically for Surveillance and forensic purpose. The unique feature of CariMuka, which enables detection of a person based on user query, makes it an outstanding product for monitoring crime-related events.

The system was released on 18 December 2019. Link to livestream.

CariMuka was one of the system that I was developing during my internship. This was a team effort with my fellow intern colleague at that time. The challenges that we faced during the initial testing was that most CCTV footage have poor video quality and some of the method that we used before has limited detection range and unable to detect faces at lower resolution since majority of CCTV footage was only shot at HD or 720p with compressed quality and different framerates. The other problem was that latency, as the system needed to detect and recognize faces in realtime at minimum 15 frames per-second.

The above picture is one of the output of the prototype, the face detection module was able to detect a face as far as 10 meter at HD quality and normal lighthing condition. To reduce the detection per frame, we implemented a tracking algorithm that uses kalman filter to compare the the new detection and the old detection to determine the detection indentification number of the face detected. The final process was the face recognition module, where at that time we use FaceNet for the prototype.

The obejctive of this app is to:

1. To allow users to open for pet adoption and adopt pets.

The app is built using Flutter with Firebase. The logo and UI/UX of the app is designed by the inhouse team. The application is developed in a limited time frame and some of the features was not able to be implemented due to time constraint.

For my part, I was tasked to develop the base of the application such as the state management functions and the main screen along with its error handling. I was also responsible for teaching and refactoring parts of the code submitted by the my teammates.
